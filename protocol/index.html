使用条款

调用本站任何接口、在本站进行投稿均代表您已经阅读并同意如下的使用条款：

一、您需要满足的条件

  1. 您不得恶意调用、攻击本站提供的任何服务。
  2. 您必须尊重百合圈的网络文化。

二、您的站点或服务的内容需要满足的条件

  1. 不违反中华人民共和国大陆地区法律。
  2. 由个人、小团体维护、运营，无盈利。
  3. 不存在任何与百合对立内容（如强烈BG向、“老婆党”等内容）。
  4. 调用频率合理（每个ip每3秒小于等于10次，浏览器直接访问每个ip每分钟小于等于5次）。

三、您的稿件需要满足的条件

  1. 不违反中华人民共和国大陆地区法律。
  2. 是您的原创或原作者不禁止分享转载。
  3. 保留原作者信息、水印（如果有）。
  4. 与百合作品相关。

四、使用本站开源代码需要满足的条件

  1. 严格遵循AGPL-3.0协议（主页代码遵循MIT协议）。
  2. 本站的台词、壁纸资源严禁商用。

五、本站的权利

  1. 拒绝不符合上述描述的服务、个人使用。
  2. 对滥用本站服务的站点与服务记忆客户端ip信息进行记录。
  3. 修改本协议。
  4. 向有关部门汇报严重违法调用者。

六、免责声明

  1. 本站提供的所有内容均来源网络、本人创作和用户投稿。如有错漏，请邮件联系我们。如有侵权，请访问这里。
  2. 本站提供的服务开放调用，不代表本站及其所有者赞成调用本站服务的服务上的内容或立场。
  3. 本站会尽可能维护服务的稳定性、可靠性，但不作任何保证。若因本站服务不可用给您带来了损失，本站概不负责。
  4. 如果您因违规使用Yurikoto源码及资源产生的任何与第三方的纠纷，Yurikoto概不负责。

七、隐私声明

  1. 本站会收集用户的ip、浏览器canvas指纹等公开数据以进行黑名单管理，但不会对外公开。合法用户的数据使用完毕后本站会自动永久删除，不作记录。但本站保留记录非法用户的信息并向有关部门汇报的权利。
  2. 本站会通过HTTP报文头收集记录调用本站服务的站点的url数据以进行审查，合法记录除了记录数量外不会对外公开任何信息，但本站保留将非法调用者的url信息向有关部门汇报的权利。
  3. 本站目前不会向用户浏览器置入或读取除了PHPSESSID外的任何Cookies信息，但是会读取HTTP报文头以进行合法性校验与审查。

八、违规处理

  1. 对于多次违规投稿但不违法者，禁用ip或浏览器指纹的投稿功能；对于违法者，向有关部门汇报ip信息。
  2. 对于违规调用本站服务但不违法的站点或服务，禁用API功能；对于违法者，向有关部门汇报相关信息。
  3. 对于调用次数超出限制者，超出限制部分返回429状态码；若有攻击嫌疑，向有关部门汇报。

九、典型违规示例

  1. 商业性质站点调用本站服务。（违规不违法）
  2. V*S、*场、黄赌毒站点或任何存在与上述内容相关内容的站点调用本站服务。（违法）
  3. “老婆党”站点调用本站服务。（违规不违法）

十、常见问题

接口调用量有什么限制？

对于网站等场景调用本站接口（正确设置HTTP报文头Referer字段，Ajax等调用方式浏览器会自动设置）每个ip每3秒可调用10次；对于浏览器等场景直接访问本站接口（未设置或未正确设置HTTP报文头Referer字段，如window.open()），每个ip每分钟可调用5次。本站检查HTTP报文头主要目的是防止滥用。

你们的服务稳定、可靠吗？

我们会尽可能使本站稳定、可靠，但不对此做任何保证。百合圈本就小众，其中的开发者更是少之又少，本站调用量也不会很大。目前本站使用原生PHP作为接口，使用MySQL存储台词与图片链接，使用Redis作访问控制，部署在某大型IDC的成都数据中心，使用付费全球CDN方案，图片资源github托管jsdelivr加速，足够应对较低的请求量。

服务端可以调用吗？

可以，但不建议调用。首先您需要正确设置HTTP报文头；您也需要注意，服务端调用时，本站获取到的是服务端ip。如果您的服务访问量较大，可能会很容易超出限制导致您的服务不稳定。

项目是否开源？

除了后台管理系统、源站校验模块等敏感部分，其他全部开源。敏感部分在版本迭代后，旧版本将开源。详见这里。

商业网站可以调用吗？

不可以，除非您提供本站用到的所有资源的商用许可。

你们限制太多了，我想自由使用，怎么办？

您可以自部署API来突破限制。请注意，自部署后不代表您可以为所欲为，您需要遵循AGPL-3.0开源协议，且本站提供的台词、壁纸资源不可商用。如果您因违规使用Yurikoto源码及资源产生的任何与第三方的纠纷，Yurikoto概不负责。
